# PC用 Firebase Authentication 仕様書

## 目的

`dev` 環境で、PC 用アプリにメールアドレス + パスワードによるログイン機能を導入する。

今回の目的は、PC からの利用を未ログインで行えない状態にし、管理画面を含む PC 側の利用者を認証前提へ移行すること。
あわせて、将来の「管理者によるアカウント発行」と「社員特定連携」の前提をそろえる。

## 対象環境

- 対象: `dev` Firebase プロジェクト
- 認証方式: Firebase Authentication の `メール / パスワード`
- 対象アプリ:
  - `dev/sinyuubuturyuu-pc/index.html`
  - `dev/sinyuubuturyuu-pc/getujitiretenkenhyou-pc/`
  - `dev/sinyuubuturyuu-pc/getujinitijyoutenkenhyou-pc/`
  - `dev/sinyuubuturyuu-pc/driver-points-kanri/`
- 対象候補の管理画面:
  - 月次タイヤ点検の管理画面
  - 月次日常点検の管理画面
  - ポイント管理画面
  - 今後追加するアカウント発行画面

## 今回の対象範囲

今回実装するのは「PC 認証導入の第1段階」とする。

### 実装するもの

- PC ランチャーのログイン画面
- ログイン状態の監視
- 未ログイン時の画面ガード
- ログアウト
- 認証エラー表示
- ログイン状態の維持
- 一般利用画面と管理画面の最低限の認証分離

### 今回はまだ実装しないもの

- 社員マスタとの自動対応付け
- 管理者 / 一般利用者の Firestore Rules 本番最終化
- パスワード再発行
- 自由な新規登録画面
- `prod` 反映
- 完全な権限ロール運用

## 利用者区分

PC 側では、最低限次の 2 区分を意識する。

### 1. 一般利用者

- 点検入力や確認を行う利用者
- ランチャーから各アプリへ入る
- 未ログインでは利用不可

### 2. 管理者

- 管理画面を使う利用者
- 将来的にアカウント発行も担当する
- 一般利用者より強い権限を持つ前提

今回の第1段階では、まずは「どちらもログイン必須」にする。
管理者専用制御は次段階で強化する。

## 画面方針

### 1. PC ランチャー初期表示

`dev/sinyuubuturyuu-pc/index.html` を開いたとき、

- 未ログインならログイン画面を表示する
- ログイン済みなら従来の PC ランチャーを表示する

### 2. ログイン画面

表示項目は次の最小構成とする。

- メールアドレス入力欄
- パスワード入力欄
- ログインボタン
- エラーメッセージ表示欄
- 認証確認中表示

必要に応じて以下を追加してよい。

- パスワード表示 / 非表示切替
- 読み込み中表示
- ログイン中ユーザー表示

### 3. ランチャー画面

ログイン後は従来の PC ランチャーを表示する。

追加する要素:

- ログアウトボタン
- ログイン中ユーザーの簡易表示
  - 例: メールアドレス

### 4. 各 PC アプリ画面

対象アプリを URL 直打ちで開いた場合も、

- 未ログインなら PC ランチャーのログイン画面へ戻す
- ログイン済みなら通常どおり表示する

### 5. 管理画面

管理画面についても、まずは

- 未ログインでは表示しない
- ログイン済みでのみ表示する

を最低条件とする。

管理者専用にする条件は次段階で整理する。

## 認証仕様

### 使用 API

- `onAuthStateChanged`
- `signInWithEmailAndPassword`
- `signOut`
- 必要に応じて `getAuth`
- 必要に応じて `initializeApp`

### ログイン成功時

- Firebase Auth のログイン状態を取得する
- PC ランチャー画面を表示する
- 既存の PC 機能はそのまま利用できる

### ログイン失敗時

ユーザーに分かる短い文言を表示する。

例:

- メールアドレスまたはパスワードが違います
- 入力内容を確認してください
- 一時的にログインできません

Firebase エラーコードは画面へそのまま出さない。

### ログアウト時

- Firebase Auth からサインアウトする
- ログイン画面へ戻す
- ランチャーや管理画面は表示しない

### ログイン状態の維持

- PC を閉じても、明示ログアウトしない限りログイン状態は維持する
- PC 再起動後も、ブラウザデータが残っていればログイン状態は保持される
- 認証確認中は未入力ログイン画面を先に見せない

## 画面遷移仕様

### A. 未ログインで PC ランチャーへアクセス

1. Auth 初期化
2. 認証確認中表示
3. 未ログイン確定後にログイン画面表示

### B. ログイン成功

1. メール / パスワードでサインイン
2. `onAuthStateChanged` でログイン済みを検知
3. PC ランチャー画面表示

### C. ログイン済みで PC アプリへアクセス

1. Auth 初期化
2. 認証確認中表示
3. 対象アプリ表示

### D. 未ログインで PC アプリ直URLへアクセス

1. Auth 初期化
2. 未ログインを検知
3. PC ランチャーのログイン画面へ戻す

### E. 未ログインで管理画面直URLへアクセス

1. Auth 初期化
2. 未ログインを検知
3. PC ランチャーへ戻す、またはログイン画面を表示する

## 実装方針

### 1. 認証処理の共通化

`dev/sinyuubuturyuu-pc/` 配下に、PC 用の認証共通 JS を作成し、ランチャーと各 PC アプリで共通利用する。

想定役割:

- Firebase Auth 初期化
- 現在ユーザー取得
- ログイン / ログアウト
- 認証状態変更監視
- 未ログイン時のガード
- 認証確認中の共通処理

### 2. UI は最初はシンプルに分離

初期段階では複雑なページ分割よりも、

- HTML 内にログイン領域
- HTML 内にアプリ領域
- 認証状態で表示切替

を優先する。

### 3. Firestore 接続前に認証確認する

既存の Firebase 連携処理は、可能な範囲で

- 認証完了後に起動する
- 未ログインなら Firestore を使う処理を走らせない

ようにする。

### 4. 匿名認証の段階廃止

PC 側も、`dev` 実装が安定したら

- 既存の匿名認証前提を段階的に廃止する
- ログイン済みユーザー前提へ寄せる

方針とする。

### 5. 管理画面は先に「ログイン必須」だけ入れる

最初から細かいロール制御を入れず、まずは

- 管理画面もログイン必須
- その後に管理者専用化

の順で進める。

## アカウント発行方針

### 基本方針

- 一般社員がスマホや PC から自由に新規登録する方式は採用しない
- アカウントは管理者が発行する
- まずは Firebase コンソールで手動発行でもよい
- 将来的に PC 管理画面へ発行機能を追加する

### 将来の管理者発行画面の役割

- 社員用メールアドレスの登録
- 初期パスワードの発行
- 利用停止 / 無効化
- 社員マスタとの対応付け

今回は仕様だけ定義し、実装は次段階とする。

## 受け入れ条件

### 基本

- 未ログインでは PC ランチャーを使えない
- 正しいメール / パスワードでログインできる
- ログイン後に PC ランチャーが表示される
- ログアウトできる
- ログアウト後は再びログイン画面に戻る

### 画面遷移

- `dev/sinyuubuturyuu-pc/index.html` 直アクセスで動作する
- 各 PC アプリ直アクセスでも未ログインなら使えない
- 管理画面直アクセスでも未ログインなら使えない

### 維持

- 再読込後もログイン状態が維持される
- PC 再起動後も必要以上に再ログインを要求しない
- 認証確認中に未入力ログイン画面が一瞬出ない

### エラー

- パスワード誤り時に適切なエラーが出る
- ネットワーク不良時に固まらない
- ログイン中の処理中表示がある

## 実装順

1. PC 用の認証共通モジュールを作る
2. PC ランチャーにログイン画面を追加する
3. ログイン / ログアウトを実装する
4. PC ランチャーの未ログインガードを入れる
5. 月次タイヤ点検 PC へ未ログインガードを入れる
6. 月次日常点検 PC へ未ログインガードを入れる
7. ポイント管理 PC へ未ログインガードを入れる
8. 管理画面へ未ログインガードを入れる
9. 認証状態維持と再起動確認を行う
10. 次段階として管理者発行画面の仕様へ進む

## 今回の完了条件

この仕様書の完了条件は次のとおり。

- `dev` の PC 用認証対象範囲が明確になっている
- ランチャー、各アプリ、管理画面の扱いが決まっている
- 一般利用者と管理者の考え方が整理されている
- 実装順が決まっている

このあと、仕様書に沿って `dev/sinyuubuturyuu-pc` の認証実装へ着手する。
