開発環境への切替を終えた作業ブランチを、本番 Firebase に戻してから main へマージするための手順です。ここでは新しいブランチは作らず、現在の作業ブランチをそのまま使います。

最重要注意

  • 開発環境へ切り替えたときの同じ作業ブランチをそのまま使います。
  • main で直接作業しません。

ブランチ名を手動で入力

開発環境へ切り替えたときに使っている現在の作業ブランチ名を、そのままプロンプトへ入れます。新しいブランチは作りません。

例: feature/dev-env-2026-04-04

3. パソコン用アプリで手動確認

  • いったんログアウトして、本番用の通常ユーザーでログインします。
  • 再読み込み後も同じユーザーが表示されることを確認します。
  • 月次タイヤ点検表 を開き、通常の入力と保存ができることを確認します。
  • 月次日常点検表 を開き、通常の入力と保存ができることを確認します。
  • ポイント管理 を開き、通常の操作と保存ができることを確認します。
  • 社員名簿 が正しく表示されることを確認します。
  • 開発環境 sinyuubuturyuu-dev に影響が出ていないことを確認します。

4. マージへ進む

上の確認が全部通ってから、その同じ作業ブランチを main へマージします。確認が終わる前にマージしません。
